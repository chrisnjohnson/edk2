Detailed Description

EDKII Platform Has Device Tree GUID

A NULL protocol instance with this GUID in the DXE protocol database, and/or a NULL PPI with this GUID in the PPI database, implies that the platform provides the operating system with a Device Tree-based hardware description. Note that this is not necessarily exclusive with different kinds of hardware description (for example, an ACPI-based one). A platform driver and/or PEIM is supposed to produce a single instance of the protocol and/or PPI (with NULL contents), if appropriate.
